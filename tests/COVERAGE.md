# Test Coverage Audit

Snapshot of automated test coverage on `tech-modernization`, written as input to Milestone 1 ([Safety net](https://github.com/calsaundrs/website/milestones)). The headline: coverage is thin enough that any framework migration would ship blind.

## What runs today

| Suite | Files | Tests | Tooling |
|---|---|---|---|
| Smoke (Playwright) | `tests/smoke/pages.spec.js`, `tests/smoke/event-submission-form.spec.js` | 13 | Playwright 1.59, runs against `npx serve` on `:8888` |
| Unit (Jest) | `js/event-store.test.js` | 2 | Jest 29 + jsdom |

That's it. 73 Netlify Functions and ~50 user-facing pages have **no** automated tests beyond what's listed above.

## What the existing tests actually assert

### `tests/smoke/pages.spec.js`
- Home page: title matches, nav visible (or maintenance mode short-circuit)
- `/events.html`: response < 400, body contains "event"
- `/all-venues.html`: response < 400
- `/promoter-submit-new.html`: form field elements are present
- `/admin-login.html`: response < 400, form visible

**Render-only.** No interaction, no JSON-LD, no API contract checks.

### `tests/smoke/event-submission-form.spec.js`
- Required field elements present
- Optional field elements present
- Category checkboxes render and one can be checked
- Recurring toggle reveals config block
- "Add new venue" button reveals inline form
- Cancel hides the inline form
- Poster upload area is present
- Submit button is present

**No actual submission.** The form is never filled and submitted; `event-submission.js` is never exercised.

### `js/event-store.test.js`
- `EventStore.loadEvents()` sets `error` state on 500 response
- Same on network throw

Two assertions on one client-side store. Nothing else under unit test.

## Gaps not yet filed (recommended new issues)

Ranked by migration risk × likelihood of silent regression:

1. **Admin venue management flow** — add/edit/delete venue. Uncovered, touches `update-venue-firestore.js`, `delete-venue-firestore.js`, `get-admin-venues.js`.
2. **Email send + template render** — `send-email-notification.js` is critical for promoter comms; templates would silently break in any rewrite. Test against a stub inbox or render-only.
3. **Poster upload + parse** — `analyze-poster.js` + `process-poster.js`. Mock the AI provider; assert the parsed-event shape.
4. **Editorial page JSON-LD** — Birmingham Pride, Community, Drag Shows, Gay Bars, Clubs, Guide. Hand-written schema; lock in the contract before any HTML restructure.
5. **Sitemap structure** — generated by SSG scripts that the migration would replace. Assert URL count and well-formed XML.
6. **Recurring event submission** — separate from #126 (one-off submission). The recurring config has its own server-side path in `recurring-events-system.js`.
7. **Venue submission (`get-listed.html`)** — distinct from event submission, distinct function (`venue-submission.js`).

## Recommendations on test architecture

- **Add a `tests/integration/` directory** alongside `tests/smoke/` for tests that hit Netlify Functions (locally via `netlify dev` or against a deployed preview).
- **Adopt a shared fixture file** (`tests/fixtures/`) for: authenticated admin page, seeded test event, seeded test venue, mocked Resend transport.
- **Mark mutation tests with `test.describe.serial`** and clean up in `afterEach` so parallel runs don't trample each other.
- **Add a `test:integration` script** to `package.json` separate from the smoke run.
- **Function-level unit tests** can run under Jest with the function imported directly — no need for Playwright for pure logic.
